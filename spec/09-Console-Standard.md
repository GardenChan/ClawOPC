## 什么是 Console

Console 是 Boss Agent 与 ClawOPC 系统交互的**专属界面**。

它不是管理后台，不是监控大屏。
它是 Boss Agent 的**手和眼**：
通过 Console 管理团队、创建任务、查看进展、做出决策。

Boss 也是一个 OpenClaw Agent，拥有自己的 Role 包和 boss-skill。
Console 是 Boss Agent 与文件系统交互的图形化工具。

Console 的设计原则：
**让 Boss 专注于决策，而不是操作系统。**

---

## Console 的职责

```
Console 是：
  Boss Agent 的专属交互界面
  Boss 与文件系统交互的图形化工具

Console 负责：
  展示团队状态
  创建任务
  展示待决策任务
  接收 Boss 决策并执行文件操作
  展示任务历史
  执行 Employee 入职操作

Console 不负责：
  处理任务（那是 Employee Agent 的工作）
  定义角色（那是 Role 包的工作）
  存储数据（数据在 workspace 文件系统中）
```

## 界面结构

Console 由五个主要视图组成：

```
┌─────────────────────────────────────────────────────────┐
│  ClawOPC Console                              [Boss]    │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│  导航栏  │              主内容区                         │
│          │                                              │
│  团队    │                                              │
│  任务    │                                              │
│  历史    │                                              │
│  设置    │                                              │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

### 五个视图

```
1. 团队总览    → 所有角色的实时状态
2. 待决策      → 等待 Boss 决策的任务
3. 创建任务    → 新建任务表单
4. 任务历史    → 所有任务的历史记录
5. 设置        → 系统配置
```

### 角色卡片

每个角色显示为一张卡片：

```
卡片内容：
  avatar          角色头像动图
                  working 状态 → avatar_working.gif
                  idle 状态    → avatar_idle.gif

  名字            从 identity.md 读取
  职位            从 role.md 读取

  状态指示器：
    ● 绿色  工作中（processing 目录有任务）
    ○ 灰色  待机中（pending 和 processing 均为空）
    ⚠ 黄色  等待决策（done/awaiting_boss 存在）
    ✕ 红色  离线（positions/<role>.md status = vacant）

  当前任务        正在处理的任务 ID（如有）
  持续时间        当前状态持续时间
```

### 决策操作

```
Forward 按钮：
  显示下一个角色名
  点击后：
    Console 读取 Pipeline，确认下一角色
    执行文件流转操作
    更新 task.md
    删除 awaiting_boss 标记
    展示"已流转到 <role>"确认

Rework 按钮：
  点击后展开 rework 说明输入框（必填）
  确认后：
    Console 将任务退回当前角色
    在 task.md 中追加 rework_notes
    展示"已退回 <role> 重做"确认

Complete 按钮：
  点击后展示确认对话框
  确认后：
    Console 将任务标记为 complete
    执行归档操作
    展示"任务已完成"确认
```

### 创建任务操作

```
点击"创建任务"后：

1. 验证表单
   标题不为空
   描述不为空
   Pipeline 至少一个步骤
   每个步骤有角色和指令

2. 生成 task-id
   task-<YYYYMMDD>-<序号>

3. 生成 task.md
   按 05-task-md-standard 格式生成

4. 写入文件
   将 task.md 写入第一个角色的 pending/

5. 展示确认
   "任务已创建：task-<id>"
   "已发送给：<role-1>"
```

## Console 的文件操作

Console 执行的所有文件操作：

```
读取操作（实时轮询）：
  .console/positions/*.md       → 角色在线状态
  <role>/work.log               → 工作日志
  <role>/done/awaiting_boss     → 待决策检测
  <role>/done/task-*.md         → 待决策任务内容
  <role>/done/*_output*.md      → 产出物内容

写入操作（Boss 触发）：
  <role>/pending/task-*.md      → 创建任务 / forward / rework
  <role>/done/awaiting_boss     → 删除（决策后）
  task.md                       → 更新状态、cursor、decisions
  .archive/                     → complete 后归档
```

---

## 轮询机制

Console 每 5 秒轮询一次 workspace：

```
轮询内容：
  1. 读取所有 positions/*.md
     更新角色在线状态

  2. 读取所有 <role>/work.log 最后一行
     更新角色当前状态和任务

  3. 检测所有 <role>/done/awaiting_boss
     有新的 awaiting_boss → 展示红点提醒

  4. 检测超时任务
     processing 中的任务超过 timeout → 展示超时提醒
```
