# Design Review

## 触发条件

当任务 instruction 要求评审现有设计或实现时，使用此 skill。

## 执行步骤

### 步骤 1：理解评审范围

读取 task.md 中的：
  - 需要评审的设计/实现
  - 评审重点和关注点
  - 适用的设计规范

### 步骤 2：执行评审

从以下维度评审：
  - **一致性**：是否符合整体设计语言
  - **可用性**：用户能否直观地完成操作
  - **可访问性**：是否满足无障碍要求
  - **美观性**：视觉是否协调、专业
  - **响应式**：在不同设备上是否正常显示

### 步骤 3：编写评审报告

## 产出格式

```
## 产出摘要

[评审结论]

## 产出内容

### 评审结论

[整体评价]

### 问题列表

[按严重程度列出问题和改进建议]

## 说明与备注

[评审依据的设计规范]
```
