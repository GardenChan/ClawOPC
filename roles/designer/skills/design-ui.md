# Design UI

## 触发条件

当任务 instruction 要求设计用户界面时，使用此 skill。

## 执行步骤

### 步骤 1：理解需求

读取 task.md 中的：
  - 需要设计的页面或功能
  - 目标用户和使用场景
  - 技术约束（如使用的框架/组件库）
  - 历史产出（如有上游角色的输出）

### 步骤 2：设计方案

制定设计方案，包括：
  - 页面整体布局
  - 核心组件设计
  - 交互流程
  - 视觉风格

### 步骤 3：编写设计说明

产出详细的设计说明文档：
  - 布局结构（Flexbox/Grid 描述）
  - 组件列表和属性
  - 间距和尺寸（具体数值）
  - 配色方案（十六进制色值）
  - 字体和排版（字号、行高、字重）
  - 交互状态（hover、active、disabled）
  - 响应式断点

## 产出格式

```
## 产出摘要

[设计方案概述]

## 产出内容

### 页面布局

[布局结构描述，含具体尺寸]

### 组件设计

[每个组件的详细设计参数]

### 配色方案

[色值列表及使用场景]

### 交互说明

[交互流程和状态变化]

## 说明与备注

[设计决策说明、替代方案、技术约束考量]

## 验收自查

- [x] 布局参数具体可实现
- [x] 配色满足对比度要求
- [x] 交互状态完整
- [x] 响应式方案明确
```
