# Write Test Plan

## 触发条件

当任务 instruction 要求编写测试方案或测试用例时，使用此 skill。

## 执行步骤

### 步骤 1：理解测试目标

读取 task.md 中的：
  - 要测试的功能/组件
  - 需求文档或实现说明（如有上游产出）
  - 验收标准
  - 技术约束

### 步骤 2：设计测试策略

确定：
  - 测试范围（测什么、不测什么）
  - 测试类型（功能/性能/安全/兼容性）
  - 测试优先级
  - 测试环境要求

### 步骤 3：编写测试用例

每个用例包含：
  - 用例 ID
  - 测试场景描述
  - 前置条件
  - 测试步骤
  - 预期结果
  - 优先级（P0/P1/P2）

覆盖范围：
  - 正常路径（Happy Path）
  - 边界条件
  - 异常输入
  - 错误处理
  - 并发场景（如适用）

## 产出格式

```
## 产出摘要

[测试方案概述]

## 产出内容

### 测试范围
### 测试策略
### 测试用例列表

| ID | 场景 | 步骤 | 预期结果 | 优先级 |
|----|------|------|----------|--------|
| TC-001 | ... | ... | ... | P0 |

### 测试环境要求

## 说明与备注

[测试策略说明、已知风险]

## 验收自查

- [x] 正常路径覆盖
- [x] 边界条件覆盖
- [x] 异常场景覆盖
- [x] 用例可执行
```
