# Data Analysis

## 触发条件

当任务 instruction 要求进行数据分析或撰写分析报告时，使用此 skill。

## 执行步骤

### 步骤 1：理解分析目标

读取 task.md 中的：
  - 要回答的业务问题
  - 可用的数据源和范围
  - 分析的时间范围
  - 报告的目标受众

### 步骤 2：分析方法设计

确定：
  - 需要的关键指标
  - 分析维度（时间/用户群/渠道等）
  - 对比基准
  - 分析方法（趋势/对比/归因/漏斗等）

### 步骤 3：执行分析

分析要素：
  - 数据概况（样本量、时间范围、数据质量）
  - 关键指标计算
  - 趋势和模式识别
  - 异常值检测
  - 相关性分析

### 步骤 4：编写报告

报告结构：
1. 核心结论（3-5 句话）
2. 关键发现（配图表说明）
3. 详细分析
4. 建议和下一步行动

## 产出格式

```
## 产出摘要

[分析结论摘要]

## 产出内容

### 核心结论

[3-5 句话的核心发现]

### 关键发现

[每个发现配数据和图表说明]

### 详细分析

[完整分析过程]

### 建议

[基于数据的可操作建议]

## 说明与备注

[数据来源、口径定义、分析局限性]

## 验收自查

- [x] 分析回答了业务问题
- [x] 结论有数据支撑
- [x] 数据来源和口径已标注
- [x] 报告对非技术读者友好
```
