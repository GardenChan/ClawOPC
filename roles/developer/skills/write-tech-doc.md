# Write Tech Doc

## 触发条件

当任务 instruction 要求编写技术文档时，使用此 skill。

## 执行步骤

### 步骤 1：理解文档需求

读取 task.md 中的：
  - 需要文档化的技术内容
  - 目标读者（开发者/运维/产品）
  - 文档类型（API 文档/架构文档/部署文档/使用指南）

### 步骤 2：组织内容

根据文档类型确定结构：
  - API 文档：接口列表、请求/响应格式、示例
  - 架构文档：系统架构、组件说明、数据流
  - 部署文档：环境要求、部署步骤、配置说明
  - 使用指南：快速开始、功能说明、常见问题

### 步骤 3：编写文档

文档规范：
  - 使用 Markdown 格式
  - 代码示例完整可运行
  - 关键概念有解释
  - 步骤说明清晰有序

## 产出格式

```
## 产出摘要

[文档概述]

## 产出内容

[完整技术文档内容]

## 说明与备注

[文档范围、假设、后续需要补充的部分]
```
