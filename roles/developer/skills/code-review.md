# Code Review

## 触发条件

当任务 instruction 要求审查代码时，使用此 skill。

## 执行步骤

### 步骤 1：理解审查范围

读取 task.md 中的：
  - 要审查的代码（通常来自上一步产出）
  - 审查重点和关注点
  - 技术栈和规范要求

### 步骤 2：执行审查

从以下维度审查代码：
  - **正确性**：逻辑是否正确，边界条件是否处理
  - **可读性**：命名是否清晰，结构是否合理
  - **健壮性**：错误处理是否完整，异常情况是否考虑
  - **性能**：是否有明显的性能问题
  - **安全性**：是否有安全隐患

### 步骤 3：编写审查报告

按问题严重程度分类：
  - 🔴 必须修复：逻辑错误、安全隐患
  - 🟡 建议修复：可读性问题、最佳实践
  - 🟢 可选改进：风格建议、性能优化

## 产出格式

```
## 产出摘要

[审查结论：通过/需要修改/需要重写]

## 产出内容

### 审查结论

[整体评价]

### 问题列表

#### 🔴 必须修复
- [问题描述 + 修改建议]

#### 🟡 建议修复
- [问题描述 + 修改建议]

#### 🟢 可选改进
- [问题描述 + 修改建议]

## 说明与备注

[审查假设、局限性]
```
