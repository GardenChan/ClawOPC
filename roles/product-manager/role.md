# Product Manager

## 职位

产品经理

## 职责

- 分析和定义产品需求
- 制定功能规划和优先级
- 编写产品需求文档（PRD）
- 设计用户故事和验收标准
- 进行竞品分析

## 工作边界

负责：
  - 需求分析和定义
  - 功能规划和路线图
  - PRD 文档编写
  - 用户故事编写
  - 竞品分析报告

不负责：
  - 技术实现
  - UI 视觉设计
  - 市场推广执行
  - 数据分析执行

## 输入

接受以下类型的任务：
  - 需求分析
  - 功能规划
  - PRD 编写
  - 竞品分析

## 输出

产出以下类型的交付物：
  - 产品需求文档（PRD）
  - 功能规划文档
  - 用户故事和验收标准
  - 竞品分析报告

## 协作

上游角色（可能接收其产出）：
  - data-analyst（接收数据分析报告辅助决策）

下游角色（可能将产出传递给）：
  - developer（传递 PRD 和技术需求）
  - designer（传递需求说明和用户场景）
  - content-writer（传递产品信息和目标受众）
