# Competitor Analysis

## 触发条件

当任务 instruction 要求进行竞品分析时，使用此 skill。

## 执行步骤

### 步骤 1：明确分析范围

读取 task.md 中的：
  - 要分析的产品领域
  - 关注的竞品（如有指定）
  - 分析重点（功能/定价/用户体验/市场策略）

### 步骤 2：执行分析

从以下维度分析：
  - 产品定位和目标用户
  - 核心功能对比
  - 优势和劣势
  - 差异化机会

### 步骤 3：总结洞察

提炼可操作的建议：
  - 我们应该学习什么
  - 我们的差异化优势在哪里
  - 有哪些未被满足的需求可以切入

## 产出格式

```
## 产出摘要

[竞品分析结论]

## 产出内容

### 竞品概览
### 功能对比
### 优劣势分析
### 差异化机会
### 建议

## 说明与备注

[分析方法、信息来源、局限性]
```
