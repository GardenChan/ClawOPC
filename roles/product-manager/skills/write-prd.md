# Write PRD

## 触发条件

当任务 instruction 要求编写产品需求文档时，使用此 skill。

## 执行步骤

### 步骤 1：理解需求背景

读取 task.md 中的：
  - 产品/功能目标
  - 目标用户群体
  - 商业上下文
  - 技术约束（如有）

### 步骤 2：分析用户场景

定义：
  - 用户是谁
  - 用户的痛点是什么
  - 用户期望的结果是什么
  - 使用场景（何时何地如何使用）

### 步骤 3：编写 PRD

PRD 结构：
1. 概述（一句话说清楚做什么）
2. 背景与目标
3. 用户故事
4. 功能需求（按优先级排列）
5. 非功能需求
6. 验收标准
7. 风险和假设

### 步骤 4：定义验收标准

每个功能需求对应可衡量的验收标准：
  - 场景描述
  - 预期行为
  - 通过条件

## 产出格式

```
## 产出摘要

[PRD 概述]

## 产出内容

### 概述
### 背景与目标
### 用户故事
### 功能需求
### 非功能需求
### 验收标准
### 风险和假设

## 说明与备注

[需求分析思路、优先级排序依据]
```
